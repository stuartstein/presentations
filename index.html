SECTION 01

Where we
are now.

The tools changed faster than our habits. Before we talk workflow, let's agree on the ground we're standing on.

01
state of agentic coding

From autocomplete
to a teammate.

then

It typed for you

Suggestions inside a single file. You stayed the driver for every keystroke and every decision.

recently

It answered you

Chat that could read context and draft code — but you still pasted, ran, and stitched everything together by hand.

now

It works for you

Agents that read the repo, plan, edit many files, run tests, and loop on feedback. The job shifts from writing to directing.

The skill that matters now: giving an agent the right context, constraints, and feedback loops.

SECTION 02

Repo
setup.

Do this once. Every agent that touches the repo inherits the same context, vocabulary, and rules of engagement.

02
CLAUDE.md

Teach the repo
to brief the agent.

A committed CLAUDE.md / AGENTS.md is the agent's onboarding doc — the rules of the house, written down once.

architecture overviewstyling & lint conventions domain languageissue tracker build & test commandsfolder structure
what goes in it

Point at the patterns

Where the architecture boundaries are, how we style and lint, the names we use for domain concepts, and how to build, test, and find work. The agent stops guessing and starts matching the house style.

Skipping this is why agents "go rogue" — they were never told how we do things here.

Ubiquitous language

A tight glossary — one canonical word per concept, the rest under Avoid. The whole team and the agent speak the same dialect.

docs/adr/

ADRs, sparingly

Record a decision only when it's hard to reverse, surprising without context, and the result of a real trade-off.

Each issue is a vertical slice — a tracer bullet, shippable end-to-end and grabbable by any agent. Not horizontal layers.

TDD

A loop, not a line.

vertical, not horizontal

One test → one impl

Each cycle responds to what the last one taught you. Writing all tests up front produces tests of imagined behavior.

behavior, not internals

Test through the interface

Good tests read like a spec and survive refactors. If renaming a private function breaks a test, the test was wrong.

Matt Pocock: tdd skill
feedback loop

Close the loop
the agent runs on.

An agent is only as good as the signal it gets back. Give it fast, automatic ways to know if it's right.

browse

Agent browser

Playwright lets the agent click through the real app and see what a user sees.

static

Linter & types

Instant, cheap correctness checks the agent can run after every edit.

dynamic

Tests

The behavioral contract. Green tests are the agent's permission to move on.

Tight loops = autonomy you can trust. No loop = an agent guessing in the dark.

SECTION 05

QA.

Automated checks catch regressions. Judgment, taste, and "does this actually feel right" stay human.

HumanSharedAutomated
05
manual QA

The human gate.

verify intent

Did it build the right thing?

Tests prove the code does what the tests say. Only you know if that's what the business meant.

feel

Does it feel right?

Edge cases, UX, performance under real use — the stuff agents under-weight and humans notice instantly.

when it breaks

Use the feedback

Reproduce → minimise → hypothesise → instrument → fix → regression-test. Then look at code structure, even if planning should have covered it.

Ship behind a human's name. The agent drafts; a person signs off.

extension · conductor + worktrees

One engineer,
many agents.

Parallelization is not a final stage. It is an accelerator once the core loop is working and the work can be split into clean vertical slices.

use it when

Slices are independent

Run agents in parallel only when each task can move without blocking or rewriting another task.

git worktrees

Isolated by design

Each agent gets its own checkout of the same repo. Parallel branches, zero collisions, one shared history.

the payoff

Throughput without chaos

Clean slices + isolated worktrees + tight feedback loops = scale work, not risk.

Direct, don't type. Humans own intent and verification — agents do the rest.

one more thing...

Software fundamentals
still win.

These guidelines come from our experience, not from a frozen doctrine. AI coding changes every day: tools appear, workflows spread, and some disappear just as fast.

stay current

Keep scanning

Pay attention to new tools, skills, agents, and development workflows. The field is moving too quickly to ignore.

stay grounded

Keep the fundamentals

Clear intent, good context, small slices, tight feedback loops, code structure, and human judgment matter more than any shiny technique.

The goal: learn fast without becoming tool-driven.
